Groovy 语法特点

发表于 2017-07-13 | 分类于 Program

体验过 Python 后，能明显感受到 Java 的短板——冗长。好在 JVM 下，还有其它优秀的语言。比较知名的，有 Kotlin、Scala。前者是谷歌 I/O 2017 钦定的 Android 开发语言，后者则备受技术大神的追捧。这篇文章中，介绍的是一门更小众的语言——Groovy。

Groovy 可以和 Java 无缝结合，同时加入了很多特性。下面是 Groovy 用法的总结：

字符串 && 可选的类型定义

可选的括号、分号

内置 setter & getter 方法

属性直接用点号获取

最后一个值当作返回值

集合 API

写在最后

目前所知，Groovy 更多地用在特定领域，例如，Gradle。除此以外，Web 框架——Grails 也是基于 Groovy，但学习资料较少，在国内属于一个小众市场，而且与SpringBoot太过相似，地位显得尴尬。

所以，如果决定用 Groovy 代替 Java 的时候，应该针对 Groovy 特性进行开发，而不仅仅是 Java 的简化版。
